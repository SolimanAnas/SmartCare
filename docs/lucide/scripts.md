# Scripts used during the migration

The migration was done with five small Python scripts in `%TEMP%\opencode\` (not committed ŌĆö they live outside the repo). They are reproducible. To re-run from scratch:

```bash
# 1. Build sprite.svg from the mapping
python build_sprite.py

# 2. Replace emojis in HTML/JS/CSS, preserving original line endings
python apply_v2.py

# 3. Repair CSS rules where my v1 accidentally dropped an <svg> tag into content: literally
python fix_css_content.py

# 4. Convert the content: url(data:image/ŌĆ”) rules into mask-image so they honor currentColor
python fix_css_mask_v3.py

# 5. Convert SVG attribute quotes from " to ' inside JS double-quoted strings
python fix_js_svg_quotes.py
```

The output of step 1 (`icons/sprite.svg`) **is** in the repo. Everything else is reproducible from `emoji-mapping.json` plus a fresh checkout.

---

## build_sprite.py

- Reads `emoji-mapping.json`.
- Collects the unique set of Lucide icon names referenced.
- HEAD-requests `https://unpkg.com/lucide-static@0.469.0/icons/<name>.svg` for each name; any that returns non-200 is dropped and reported.
- Parses each response, strips the outer `<svg ŌĆ”>` wrapper, and emits the inner geometry as a `<symbol id="<name>" viewBox="0 0 24 24" fill="none" stroke="currentColor" stroke-width="2" stroke-linecap="round" stroke-linejoin="round">`.
- Writes `icons/sprite.svg` (style="display:none" on the root so the sprite itself never renders).

The result is 122 symbols in 42 KB. Some of the inner geometry contains Lucide's `<!-- @license ŌĆ” -->` HTML comment, which is stripped before writing.

## apply_v2.py

Walks every `.html`, `.htm`, `.js`, `.mjs`, `.css` under the repo (skipping `node_modules`, `venv`, `_archive`, `.github`, etc.) and:

1. Reads the file as **bytes**, not text ŌĆö important so encoding stays UTF-8 and line endings are visible.
2. Detects the dominant line ending (`\n` vs `\r\n`) by counting.
3. Normalizes the content to LF for processing, runs:
   - For `.css`: `re.sub(r"content\s*:\s*(['\"])([^'\"]+?)\1", css_repl, ŌĆ”)` ŌĆö when the captured string is in the emoji-to-Lucide map, emit a `mask-image` URL pointing at the matching sprite symbol's geometry.
   - For everything else: `emoji_re.sub(repl, ŌĆ”)` ŌĆö replace each matched emoji with an inline `<svg class="lucide" width="1em" height="1em" aria-hidden="true" focusable="false"><use href="<sprite_path>#<lucide-name>"/></svg>`. Sprite path is computed from the file's depth: `icons/sprite.svg` for root, `../` ├Ś depth otherwise.
4. Re-applies the original line ending before writing.

A previous version of this script used `Path.read_text` / `Path.write_text`, which silently converted all line endings to CRLF on Windows. The byte-based version preserves them, so the resulting git diff only shows the actual emojiŌåÆSVG content changes.

The `emoji_re` regex is built from the sorted keys of the mapping, longest first, so multi-codepoint emoji like `Ōä╣’ĖÅ` and `ŌØż’ĖÅ` match before the bare `Ōä╣` or `ŌØż`.

## fix_css_content.py

First pass of the CSS repair. The first version of `apply_v2.py` had a bug: it ran the emoji regex *before* the CSS `content:` rule regex. So a rule like `content:'­¤Üæ'` was first converted to `content:'<svg ŌĆ”></svg>'`, then the CSS regex saw it, found no emoji in the captured string, and left the now-invalid `content:'<svg ŌĆ”></svg>'` in place. Browsers tolerate that badly ŌĆö `content:` with an HTML-looking string shows nothing.

`fix_css_content.py` finds those literal `<svg class="lucide"ŌĆ”><use href="ŌĆ”#ICON"/></svg>` strings sitting inside `content:'ŌĆ”'` and rewrites the entire `content:` rule into a `url("data:image/svg+xml;charset=utf-8,ŌĆ”#ICON")` URL pointing at the same icon.

## fix_css_mask_v3.py

`content: url("data:image/svg+xmlŌĆ”")` works, but the icon's stroke is **baked into the SVG** (it was rendered with `fill="black" stroke="black"`), so it can't pick up `currentColor` from the page. Theme switches don't affect it.

This pass rewrites each `content: url(ŌĆ”)` rule into a properly-themed `mask-image` form:

```css
.cat-header::before{
  content:"";
  display:inline-block;
  width:1em; height:1em;
  background-color:currentColor;
  -webkit-mask:url("data:image/svg+xml;ŌĆ”#ambulance") no-repeat center/contain;
          mask:url("data:image/svg+xml;ŌĆ”#ambulance") no-repeat center/contain;
  vertical-align:-.125em;
}
```

The script matches by reading the URL fragment (`%23NAME`) that was already in the data URL after `fix_css_content.py`. It then:

1. Looks the icon up in `icons/sprite.svg` to get the geometry.
2. Wraps it in a minimal SVG with `fill="black" stroke="black"` (required for the mask).
3. URL-encodes the result and emits the rule above.

Selectors handled: `.<name>::before` and `.<name>::after`.

## fix_js_svg_quotes.py

The replacement string generated by `apply_v2.py` used double quotes for SVG attributes (`<svg class="lucide" ŌĆ”><use href="ŌĆ”"/></svg>`). Inside a JS double-quoted string that becomes:

```js
shortTitle: "<svg class="lucide" width="1em" ŌĆ” > Full Index",
```

ŌĆö a syntax error (`Unexpected identifier 'lucide'`).

This pass rewrites every complete `<svg class="lucide" ŌĆ”><use href="ŌĆ”"/></svg>` block inside `.js` files to use single-quoted attributes:

```js
shortTitle: "<svg class='lucide' width='1em' ŌĆ” > Full Index",
```

The match is global and attribute-aware, so even SVGs with extra attributes (`stroke-width`, `aria-hidden`, etc.) get all their quotes normalized.

`content/*.js` had the most occurrences (the medication index, the chapter content generators, the full-index page). In total 581 SVG blocks across 35 JS files were rewritten.

## Verification helpers

A few throwaway scripts were used to sanity-check the work and are not part of the migration itself:

- `scan_emojis.py` ŌĆö re-walks the repo for emoji characters and counts them. Confirms the result of every pass.
- `find_remaining.py` ŌĆö reports any emoji still in HTML/JS/CSS after the mapping. Should be empty (excluding the intentionally-kept color/copyright/decorative ones).
- `validate_sprite.py` ŌĆö parses `icons/sprite.svg` as XML and confirms each referenced icon id is present.
- `validate_refs.py` ŌĆö scans every HTML/JS/CSS file for `sprite.svg#ŌĆ”` references and warns about any id that isn't in the sprite.
- `validate_icons.py` ŌĆö confirms each candidate Lucide name returns 200 from `https://unpkg.com/lucide-static@0.469.0/icons/<name>.svg`.

## Inputs

All five scripts read the same JSON map: `emoji-mapping.json`. To add a new emoji later:

1. Append an entry to that file: `"­¤ÜĆ": { "lucide": "rocket", "category": "ui" }`.
2. (Optional) validate the icon name: `python -c "import urllib.request; print(urllib.request.urlopen('https://unpkg.com/lucide-static@0.469.0/icons/rocket.svg').status)"`.
3. Re-run `apply_v2.py`, then `fix_css_content.py`, then `fix_css_mask_v3.py`, then `fix_js_svg_quotes.py`.
4. Run `scan_emojis.py` to confirm the new icon is gone from the source files.
